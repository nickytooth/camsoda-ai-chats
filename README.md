# Victoria — AI Girlfriend Chat Demo

A web-based AI girlfriend chat app with two modes: **Story** and **Sexting**. Features a dark Candy.ai-inspired UI, real-time WebSocket chat, a single always-open persona, two-tier memory (STM + LTM with vector search), time-of-day awareness, and photo exchange.

---

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Frontend | Next.js 16 + React + Tailwind CSS |
| Backend | Python FastAPI + uvicorn |
| Real-time | WebSocket (native) |
| Chat / Story LLM | xAI Grok (generates every reply) |
| Generation fallback | Google Gemini Flash (when Grok fails) |
| Classifier / Summarizer | Google Gemini Flash |
| Embeddings | OpenAI text-embedding-3-small |
| Vision (user photos) | Grok Vision |
| Database | PostgreSQL (asyncpg) |
| Language | Python 3.13+ / TypeScript |

---

## Modes

### Sexting Mode
- Message batching (debounce window), then process together
- Always-open persona — every reply is generated by Grok
- She sends photos naturally during conversation
- Time-of-day + weather awareness (Miami timezone) affects mood
- User can send photos (analyzed via Grok Vision)

### Story Mode
- Follows a predefined story arc (`stories/victoria_story.yaml`)
- Grok only — no model switching
- One message at a time (send → wait → respond)
- `*Italic roleplay actions*` rendered in the UI
- Story progress tracked per user in the DB

## Setup

### 1. Install

```bash
git clone <repo-url>
cd camsoda-ai-chats

# Backend
python -m venv .venv
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # Linux/Mac
pip install -r requirements.txt

# Frontend
cd frontend
npm install
cd ..
```

### 2. Configure

```bash
cp .env.example .env
```

Edit `.env`:

| Variable | Description |
|----------|-------------|
| `ANTHROPIC_API_KEY` | Anthropic API key for Claude (SFW chat) |
| `XAI_API_KEY` | xAI API key for Grok (NSFW chat + Story mode + Vision) |
| `GOOGLE_API_KEY` | Google AI API key for Gemini (classifier + summarizer) |
| `OPENAI_API_KEY` | OpenAI API key (embeddings only) |
| `DATABASE_PATH` | SQLite DB path (default: `data/bot.db`) |
| `MIN_RESPONSE_DELAY` | Min reply delay in seconds (default: `3`) |
| `MAX_RESPONSE_DELAY` | Max reply delay in seconds (default: `10`) |
| `SERVER_PORT` | Backend port (default: `8000`) |

### 3. Run

```bash
# Terminal 1 — Backend
python -m uvicorn server.app:app --host 0.0.0.0 --port 8000

# Terminal 2 — Frontend
cd frontend
npm run dev
```

Open `http://localhost:3000` in your browser.

## Architecture

### Message Flow

```
User sends message (WebSocket)
       │
       ├── Story mode ──────────────────────────┐
       │   Direct processing, Grok only          │
       │   Story context injected from YAML      │
       │                                         │
       └── Sexting mode ─────────────────────────┤
           Batched (3-10s collect window)         │
           SFW/NSFW classification                │
           SFW → Claude / NSFW → Grok            │
                                                  │
                                                  ▼
                                          Add to STM (mode-aware)
                                          Maybe summarize → LTM
                                          Maybe compact LTM
                                          Retrieve LTM (vector search)
                                                  │
                                                  ▼
                                          Build prompt:
                                          [persona + time context + story context]
                                          [LTM memories + user facts]
                                          [STM conversation turns]
                                                  │
                                                  ▼
                                          Generate response
                                          Split into message bubbles
                                                  │
                                                  ▼
                                          WebSocket events:
                                          typing_start → delay → typing_end
                                          message (× N bubbles, with pauses)
```

### Memory System

- **STM**: Last 18 turns per mode (sexting and story have separate history)
- **LTM**: Structured memories with vector embeddings, importance scores, recency
- **Facts**: Key-value user facts (name, preferences) — shared across modes
- **Summarization**: Every 18 user turns, Gemini summarizes oldest 10 messages → LTM
- **Compaction**: At 500+ LTM entries, Gemini deduplicates and merges
- **Retrieval scoring**: similarity (0.5) + importance (0.3) + recency (0.2), top 5

### API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/ws/chat` | WebSocket | Real-time bidirectional chat |
| `/api/history/{mode}` | GET | Chat history for a mode |
| `/api/story/chapter` | GET | Current story chapter |
| `/content/{filename}` | GET | Static content files (photos) |

## Database Schema

| Table | Purpose |
|-------|---------|
| `messages` | STM — messages per user per mode (`sexting` / `story`) |
| `memories` | LTM — structured memories with embeddings |
| `user_facts` | Key-value facts about the user |
| `sent_content` | Tracks content sent (dedup) |
| `story_progress` | Current chapter per user |
| `engagement_state` | NSFW counters, push tracking |

## Tunable Constants

| Constant | File | Default | Description |
|----------|------|---------|-------------|
| `STM_MAX_TURNS` | `config.py` | 18 | Turns before STM → LTM summarization |
| `STM_SUMMARIZE_BATCH` | `config.py` | 10 | Messages summarized per batch |
| `LTM_TOP_K` | `config.py` | 5 | Memories retrieved per message |
| `LTM_COMPACTION_THRESHOLD` | `config.py` | 500 | Entries before compaction |
| `SEXTING_DEBOUNCE_SECONDS` | `config.py` | 5 | Debounce before she replies (seconds) |
| `SOFT_PUSH_THRESHOLD` | `engagement.py` | 8 | NSFW messages before content hint |
